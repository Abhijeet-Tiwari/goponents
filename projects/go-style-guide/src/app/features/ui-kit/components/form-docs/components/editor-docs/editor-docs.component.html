Setup

For any implementation of an Editor, we use TinyMCE. However, Goponents provides an extended version of the TinyMCE Editor component to handle default values and theming capabilities that are in line with the styling of Goponents. This is the go-editor component.

Step 2: Setup Reference to TinyMCE & Theme Files

You must tell Angular to include the TinyMCE scripts and theme files in the build as part of the public assets. To do so you can add an entry to the build.options.assets section of the angular.json file.

Step 3: Setup Reference to TinyMCE Files

Wherever you initially setup your GoConfigInterface on the GoConfigService, you need to define the tinyMceConfig.base_url property on the config object. This refers to the location in the URL to look for TinyMCE and is relative to the domain of your application.

Adding Languages (Optional)

TinyMCE has a feature to support multiple languages, to enable this support in the go-editor you must provide the language code and Javascript file that holds the translations. You can do this in the GoConfigInterface on the GoConfigService.

Step 4: Setup Lazy Loading of the Module

In the module that you're rendering the go-editor (or in a shared module) you must tell Angular where to load the TinyMCE script from when a page that needs it loads. To do this, first add the GoEditorModule to the list of imports for the module, then add an entry to the providers array. The useValue property on the entry refers to the location in the URL to look for the TinyMCE Javascript file and is relative to the domain of your application.

Step 5: Render the Component

Once all of the previous steps have been completed you can render the go-editior. The only required property on the go-editor is the formControl binding.

Further Customization

The go-editor component directly extends the TinyMCE Angular component, therefor you have access to all of the bindings available on TinyMCE itself. If you need further customization outside of the documentation provided here please refer to the TinyMCE documentation.
